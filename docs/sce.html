Coding Environment

We use Defra’s Scientific Coding Environment which allows us to write and run code on Amazon Web Service’s EC2 virtual machines, and store our data using Amazon Web Services S3. Speak to the SCE admins (Ope and Jake as of September 2023) if you are not set up on the SCE. Guidance documents can be found at on Sharepoint.

S3

S3 is accessed through the AWS console. When we upload, modify or delete data on S3, this must be recorded in the Data Log.

All binned waste data should be stored in the ‘moduleB2’ folder at: s3://s3-ranch-043/moduleB2/.

Scripts for reading and writing to AWS can be found in the useful scripts repo. Here is an example of reading an Excel file:

excel_data <- aws.s3::s3read_using(
  FUN = openxlsx::read.xlsx,
  object = "test_folder/test_data.xlsx", # this file exists, but replace with your own filepath
  sheet="Sheet1",
  fillMergedCells = TRUE,
  bucket = "s3-ranch-043")
