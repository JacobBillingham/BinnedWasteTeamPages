How to edit Github code

Once you have cloned a repo it is likely you will want to make some changes to the code in it. To do that follow these steps:

  1. For QA purposes we do not edit the main or dev branches directly, so you will need to create a branch of your own. To do that in R Studio click the ‘New Branch’ button in the Git window.

  1. You should then name your branch the same as the Jira ticket you are working on, for instance “B1-20 Extract and reformat RFI data”. Then click ‘Create’. The branch name next to the ‘New Branch’ button should now have the name of your branch.

  2. Now is time for you to make your edits! Make any changes you nee, following the coding standards, and frequently ‘commit’ (this is explained in the next step).

  3. When you make and save some changes Git will notice, so any changed files will appear in the Git window. Committing is like telling git those changes are ready to be uploaded to Github (which will happen in the next step). To commit:

    1. Tick the box in the Git window for the files you wish to commit.
    2. Click the commit button.

    1. In the window that opens, review your changes are as intended.
    2. Then add a ‘Commit message’ explaining what you have done, and click commit underneath.
    3. Close the window and go back to making changes.
  • It is good practice to commit frequently so your changes are logged incrementally on Github.

  1. Once you have committed all the changes you need, we can ‘push’ them to Github. You should do this once you have finished a ticket and whenever you stop working on the code e.g. for a meeting, to have lunch or at the end of the day. Follow these steps:

    1. Firstly it is good practice to ‘pull’ to check no-one has changed our branch whilst we were working. Click the ‘Pull’ button and if prompted enter your Github username but instead of our password we need to use the token we generated in step 1 of ‘How to clone a repository’. Close any the Git pull pop-up.

    1. Now we are ready to ‘push’, click the ‘Push’ button and once again if prompted enter your Github username but instead of our password we need to use the token we generated in step 1 of ‘How to clone a repository’.

    1. You have now successfully pushed your changes to Github.
